# About this Guide

This document is intended as a reference guide, to provide information on the available Thredd XML reports.

It describes the Transaction XML Data Schema elements, sub-elements and attributes.

## Target Audience

Technical team(s) responsible for the handling and processing of the Thredd transaction XML files.

You should have reasonable knowledge of XML/XSD and of the Cards industry.

## What's Changed?

If you want to find out what's changed since the previous release, see the [Document History](../Document_History.htm) section.

## How to use this Guide

If you are new to Thredd and want to understand when and how XML files are provided to you, the types of XML files available and how they can be used,

see the [Introduction](Introduction.htm) and [Transactional Data Files](Transactional_Data_Files.htm) topics.

To view an example of a Transaction XML file, see [Transaction XML Example](../Reference/Transaction_XML_Example.htm).

To view the current data schema, see [Transaction XML Schema](../Reference/Transaction_XML_Schema.htm).

For upcoming/future versions of the Transaction XML schema, see the [XSD Schemas](https://docs.thredd.ai/Downloads.htm) section on the Documentation Portal.

For information about the record types and fields in the Transaction XML file, see [Primary Elements](../Schema/Primary_Elements.htm).
