# 4.8 Integrating the Thredd Service

Once you have signed up to Thredd and completed your issuer and Thredd product setup forms, you are now ready to start integrating your systems to Thredd. Below is a high-level summary of what your development team will need to do:

| 창혞혳 | Explore the Thredd API in our Sandbox environment. |
| --- | --- |
| 창혞혳 | Request access to Thredd systems and test account credentials for your dedicated account. |
| 창혞혳 | Integrate the Thredd API into your front-end apps/back-end systems. |
| 창혞혳 | Use the Card Transaction System to put through transaction simulations. |
| 창혞혳 | If using the Thredd External Host Interface (EHI), configure your systems to be able to receive and process EHI messages. |
| 창혞혳 | If using the Thredd daily XML reports, configure your systems to process these reports. |
| 창혞혳 | Integrate your back-end authorisation and message processing engine and databases to update card transaction and balance details, based on messages received via EHI and XML reports. |
| 창혞혳 | Train staff members on the use of Thredd systems and how to support cardholders. |
| 창혞혳 | Pilot test your initial cards internally before rolling out to your customers. |

For more information about the steps in integrating the Thredd service, see the Thredd [Getting Started Guide](https://docs.thredd.ai/gettingstarted/Content/GSHome.htm).
