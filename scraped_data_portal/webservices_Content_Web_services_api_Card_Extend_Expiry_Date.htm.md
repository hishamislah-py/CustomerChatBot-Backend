# Card Extend Expiry Date

API: `Ws_ExtendExpiry`

This web service enables you to change the Thredd expiry date of a card.

The Thredd expiry refers to the expiry date held by Thredd, which is calculated based on the Thredd validity period from the [ProductMaster![Closed](../../Skins/Default/Stylesheets/Images/transparent.gif) Card product-level master record](#). This web service does not allow you to change the physical expiry date embossed on the card. The calculation of this date is based on the card validity period from the [SchemeMaster![Closed](../../Skins/Default/Stylesheets/Images/transparent.gif) Card scheme-level master record](#).

The use of this web service is primarily when a scheme operates a shorter validity period within Thredd than the embossed expiry validity period. In this case you can extend the cardholderâs Thredd-held expiry period and potentially charge a fee.

Changing the expiry date also updates the card status to '*00*' (*All Good*).

You cannot extend the expiry date of a card which has been set to an irreversible status (such as *Stolen* or *Destroyed*); see [Status Codes](../Reference/Status_Codes.htm). If the card is in an irreversible status, this web service returns the [Action Code](../Reference/Action_Codes.htm) 654.
